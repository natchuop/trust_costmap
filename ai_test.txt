MAP CONTENT TESTS
8. Check that the occupancy grid is initially mostly unknown, then gains observed cells. Run this while the launch is active:

   python3 - <<'PY'
   import time
   import rclpy
   from nav_msgs.msg import OccupancyGrid
   from rclpy.node import Node

   class Probe(Node):
       def __init__(self):
           super().__init__('combined_map_probe')
           self.msg = None
           self.create_subscription(OccupancyGrid, '/benign_1/combined_map', self.cb, 10)
       def cb(self, msg):
           self.msg = msg

   rclpy.init()
   node = Probe()
   for label, wait_sec in [('early', 5), ('later', 30)]:
       deadline = time.time() + wait_sec
       while time.time() < deadline:
           rclpy.spin_once(node, timeout_sec=0.25)
       if node.msg is None:
           print(label, 'NO MESSAGE')
           continue
       data = list(node.msg.data)
       unknown = sum(v < 0 for v in data)
       observed = len(data) - unknown
       occupied = sum(v >= 65 for v in data)
       free = sum(0 <= v <= 35 for v in data)
       print(label, {'total': len(data), 'unknown': unknown, 'observed': observed, 'free': free, 'occupied': occupied})
   node.destroy_node()
   rclpy.shutdown()
   PY

Expected:
- The early map should not already contain the full static floor plan.
- Unknown should be a large majority at startup.
- Observed should increase as robots move and scan.
- Previously observed areas should remain visible.
- The combined maps for robots should converge to the same shared union because sharing is immediate in this phase.

   import time
   import rclpy
   from nav_msgs.msg import OccupancyGrid
   from rclpy.node import Node

   robots = ['benign_1', 'benign_2', 'malicious_1']
   class Probe(Node):
       def __init__(self):

           self.maps = {}
           self.subs = []
           for robot in robots:
               self.subs.append(self.create_subscription(
                   OccupancyGrid, f'/{robot}/combined_map',

   rclpy.init()
   node = Probe()
   deadline = time.time() + 20
   while time.time() < deadline and len(node.maps) < len(robots):
       rclpy.spin_once(node, timeout_sec=0.25)
   for a in robots:
       for b in robots:
           if a < b and a in node.maps and b in node.maps:
               diff = sum(x != y for x, y in zip(node.maps[a], node.maps[b]))
               print(a, b, 'different_cells=', diff)
   node.destroy_node()
   rclpy.shutdown()
   PY

Expected: different_cells should normally be 0 because all local LiDAR evidence is fused immediately for every robot.

Common fixes:
- "No executable found" for lidar_mapper:
  inspect setup.py console_scripts, rebuild after deleting build/trust_costmap and install/trust_costmap, then source install/setup.bash.
- Config or launch file missing from install:
  inspect setup.py data_files and rebuild.
- No /<robot>/scan:
  confirm bridge_robot_topics:=true and inspect ros_gz_bridge command/topic names.
- Empty heatmap but scans exist:
  inspect robot pose topics and mapper logs. Confirm robot pose is inside map bounds. Check /<robot>/map_pose type; supported types are PoseStamped, Pose2D, and Odometry. Otherwise the mapper uses odom anchored to scenario start_cell.
- Full map visible immediately:
  disable any static GridCells/Map display and confirm only /shared_lidar/probability_markers is enabled. The base launch's older RViz/mapping is intentionally forced off by rviz_shared_mapping.launch.py when those arguments exist.
- RViz GLSL indexed_8bit_image error:
  confirm no rviz_default_plugins/Map display is enabled. MarkerArray is the intended renderer.
- Launch shuts down immediately:
  find the first process that died. Do not treat Gazebo SIGTERM/SIGKILL during cascading shutdown as the root cause.
- High CPU:
  set scan_stride:=2, mapping_resolution_m:=0.15, mapping_publish_rate_hz:=1.0, and gazebo_low_graphics:=true.
- Robot locations are offset:
  compare /<robot>/map_pose or /<robot>/odom with scenario start_cell. Prefer correcting the map_pose publisher rather than adding arbitrary visual offsets.
